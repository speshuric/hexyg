- Исправлены критические ошибки в грамматике:

  - Убран символ `:` из UNQUOTED_VALUE для корректного парсинга struct полей
  - Изменена логика валидации: ambiguity warnings допустимы, критичны только syntax errors
- Добавлена документация команды `cargo test -p hexyg-grammar-check` в README.md и README.ru.md

### 2026-01-29
- Реализовано ядро `hexyg-core`:
  - Структура `Config` с поддержкой всех основных опций (endian, line_length, block_length, preview)
  - Функция `bin_to_hex` для конвертации бинарных данных в hex-текст
  - Функция `hex_to_bin` для конвертации hex-текста в бинарные данные
  - Поддержка комментариев, директив, ASCII preview и аннотаций при парсинге
  - Покрытие unit-тестами (6 тестов)
- Реализован CLI `hexyg-cli`:
  - Парсинг аргументов командной строки через clap
  - Команды `bin-to-hex` и `hex-to-bin`
  - Поддержка stdin/stdout и файлового ввода-вывода
  - Опции: `--input`, `--output`, `--endian`, `--line-length`, `--block-length`, `--no-preview`
  - Успешно протестирован на эталонном файле `tests/corpus/valid/01_reference.hex`
- Исправлен CLI согласно спецификации:
  - Изменены команды на `--bin-to-hex` и `--hex-to-bin` (вместо subcommands)
  - Добавлены все опции из спецификации: `--from`, `--to`, `--start-address`, `--address-size-step`, `--byte-separator`, `--block-separator`, `--check`, `--option`
  - Базовая функциональность (`--address-size`, `--line-length`, `--block-length`, `--repeat-address`) полностью работает
  - Остальные опции распарсены, но логика будет реализована позже
- Объединена документация из `docs/USAGE.md` и `specs/02_options.md` в единый файл `specs/02_options.md` с примерами использования и статусом реализации
- Добавлены короткие варианты опций: `-i` (--input), `-o` (--output), `-?` и `-h` (--help)
- Исправлено название значения address-size: `arbitrary` → `stretch`
