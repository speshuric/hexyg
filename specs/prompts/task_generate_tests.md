# Task: Generate Unit Tests

Для предоставленного кода:
1. Проанализируй все граничные случаи (пустой ввод, переполнение, неверные данные).
2. Используй библиотеку `rstest` (если применимо) или стандартный `#[test]`.
3. Если тестируешь парсер, используй примеры из `@specs/01_grammar_spec.md`.
4. Не удаляй существующий код, только добавляй модуль `mod tests`.
